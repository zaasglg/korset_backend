# Настройка CategoryResource для иерархии категорий

## Что было изменено:

### 1. CategoryResource (Основные категории)
- **Фильтрация**: Показывает только категории с `parent_id = null`
- **Колонки**: Добавлен счетчик подкатегорий
- **Форма**: Убрано поле выбора родительской категории
- **Relationship**: Добавлен ChildrenRelationManager для управления подкатегориями

### 2. ChildrenRelationManager
- Управление подкатегориями внутри родительской категории
- Создание, редактирование и удаление подкатегорий
- Простая форма без поля parent_id (автоматически устанавливается)

### 3. Структура навигации
- Группа "Каталог" для всех категорий
- Основные категории имеют приоритет 1

## Как это работает:

1. **Основной экран CategoryResource**:
   - Показывает только родительские категории (parent_id = null)
   - Отображает количество подкатегорий для каждой категории
   - Есть описание, объясняющее логику

2. **Управление подкатегориями**:
   - Откройте любую основную категорию для редактирования
   - Перейдите на вкладку "Подкатегории"
   - Там можно создавать, редактировать и удалять подкатегории

3. **Автоматическая связь**:
   - При создании подкатегории через RelationManager автоматически устанавливается parent_id
   - Подкатегории не отображаются в основном списке категорий

## Преимущества такого подхода:

✅ Чистый интерфейс - только основные категории в списке
✅ Логичная структура - подкатегории внутри родительских
✅ Удобное управление иерархией
✅ Счетчики для быстрой оценки структуры
✅ Сохранена возможность работать со всеми категориями отдельно

## Дополнительные возможности:

Если нужен доступ ко всем категориям одновременно, можно использовать AllCategoriesResource с фильтрами по уровню вложенности.
